                        SECURITY & ISOLATION
═══════════════════════════════════════════════════════════════════════════════

Multi-Tenant Isolation:
├─ business_id in all tables
├─ All queries filter by business_id
├─ RPC functions enforce business_id
└─ Users can only see their business's data

Role-Based Access Control:
├─ 8 roles with hierarchy levels (0 = admin, 99 = employee)
├─ Role → Modules (role_functions table)
├─ Role → Actions (role_function_actions table)
├─ Permission checking via RPC functions
└─ Frontend displays only accessible modules

Data Protection:
├─ UUID user ID format (no sequential IDs)
├─ Passwords stored (plaintext - should be hashed in production)
├─ RPC functions use SECURITY DEFINER
├─ Prepared statements prevent SQL injection
└─ CORS enabled for frontend communication
