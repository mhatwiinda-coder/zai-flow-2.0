💡 HOW IT WORKS BEHIND THE SCENES

Dynamic Sidebar System:
  ┌──────────────────────────────────────────────────────────────┐
  │ 1. Page loads (any ERP page)                                 │
  │ 2. DOMContentLoaded event fires                              │
  │ 3. sidebar-manager.js initializes                            │
  │ 4. Get user from localStorage                                │
  │ 5. Call RPC: get_user_accessible_modules(user_id, biz_id)   │
  │ 6. RPC queries:                                              │
  │    - user_roles (get user's roles)                           │
  │    - role_functions (get module for each role)               │
  │    - functions (get module details: name, icon, url)         │
  │ 7. Return: Array of modules user can access                  │
  │ 8. JavaScript renders sidebar with only those modules        │
  │ 9. Sidebar is ready - user clicks modules to navigate        │
  └──────────────────────────────────────────────────────────────┘

Role Assignment System:
  ┌──────────────────────────────────────────────────────────────┐
  │ 1. Admin opens User Management                               │
  │ 2. Page loads, queries all users in business                 │
  │ 3. Query user_roles to see current assignments               │
  │ 4. Admin selects role from dropdown                          │
  │ 5. Admin clicks "Assign Role"                                │
  │ 6. Call RPC: assign_user_role(user_id, admin_id, role_id)   │
  │ 7. RPC inserts into user_roles table                         │
  │ 8. Returns success message                                   │
  │ 9. Admin UI updates to show new role                         │
  │ 10. Next time user logs in, gets new modules                 │
  └──────────────────────────────────────────────────────────────┘

🎓 UNDERSTANDING THE SYSTEM

Question: Why do users need to login again to see new modules?
Answer: The sidebar is generated when the page loads. It reads from
        localStorage which contains the old user data. On next login,
        the server returns fresh user_roles and modules load correctly.
        (Could be optimized with WebSockets for real-time updates)

Question: Can a user manually access /accounting.html if they're not allowed?
Answer: Yes, they can type the URL. But:
        1. Sidebar won't show the link (no access icon visible)
        2. Page loads with empty sidebar (sidebar-manager runs)
        3. RPC returns 0 accounting functions for non-accounting user
        4. Page functionality requires RPC access which user lacks
        5. Operations will fail with permission error from RPC

Question: How does multi-tenancy work?
Answer: All queries filter by business_id. Users can only see and modify
        other users in their same business. Different businesses are
        completely isolated in the database.

📞 SUPPORT

Issue: User still shows 0 modules after role assignment
Fix: 1. User must logout and login again
     2. Page must reload to run sidebar-manager.js
     3. Check user_roles table has entry
     4. Check role_functions table has module mapping

Issue: Sidebar shows modules but user can't access them
Fix: 1. Check role_function_actions table
     2. RPC might not have action permission
     3. Check user role hierarchy
     4. Verify RPC calls in JavaScript console (F12)

Issue: Admin Panel not showing for admin user
Fix: 1. Check user_role in localStorage
     2. Must be exactly 'admin' (case-sensitive)
     3. Refresh page after role assignment
     4. Check sidebar-manager.js is loaded (F12 Network tab)
